# Security and responsible use

X Media CI is a local-first archiving toolkit. It is intended for personal research, documentation, and maintainer workflows involving content the user is already authorized to access.

## Responsible-use policy

Do not use this project to:

- bypass X/Twitter login, access controls, paywalls, private-account restrictions, or rate limits;
- operate a public scraping service;
- redistribute third-party media without permission;
- archive private, sensitive, or personal data without authorization;
- evade deletion requests, copyright restrictions, or platform rules;
- automate engagement, spam, impersonation, or platform manipulation.

Use the tool only with your own browser session and for content you are allowed to view.

## Platform and copyright considerations

X Media CI can preserve text and media locally. That does not grant rights to republish or redistribute third-party content. Users are responsible for:

- respecting platform terms;
- respecting copyright and licenses;
- removing archived material when required;
- handling personal data appropriately;
- avoiding public release of captured media unless they have permission.

The recommended default is to keep archives private and local.

## Local server warning

The FastAPI server under `tools/server/` is designed for local networks and personal devices. By default it is convenient, not hardened for public internet exposure.

Before exposing it outside a trusted machine or LAN, add:

- authentication;
- authorization;
- TLS termination;
- request size limits;
- path traversal review;
- audit logging;
- firewall restrictions.

Do not bind the server to a public interface on a VPS without hardening it first.

## Browser profile and cookies

The Playwright capture workflow may use `--user-data-dir` to reuse a browser session. That directory can contain cookies and other sensitive browser state.

Recommendations:

- store it outside the repository;
- add it to `.gitignore`;
- do not share it;
- delete it if a machine is compromised;
- use a dedicated browser profile for capture workflows.

## Archive data privacy

Captured archives may contain:

- third-party images and videos;
- usernames and profile metadata;
- timestamps;
- replies and thread context;
- notes and exports generated by the user.

Treat archive directories as private data unless explicitly prepared for sharing.

## Reporting security issues

If you find a vulnerability in this project, please open a private report if GitHub Security Advisories are enabled, or contact the maintainer directly before public disclosure.

When reporting, include:

- affected version or commit;
- reproduction steps;
- expected and actual behavior;
- impact;
- suggested mitigation if known.

## Scope

In scope:

- path traversal or arbitrary file read/write in the local server;
- unsafe default server exposure;
- command injection in export/job APIs;
- accidental credential leakage;
- archive validation bugs that can overwrite unrelated files.

Out of scope:

- bypassing X/Twitter access controls;
- attacks against X/Twitter itself;
- large-scale scraping requests;
- issues caused by publishing private archives intentionally.
