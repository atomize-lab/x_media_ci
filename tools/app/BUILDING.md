# Building the x_media CI Flutter app (Android 14+)

This project uses a single Flutter codebase to target **Win11 / Ubuntu22 / Android 14+**.
You only need the Flutter SDK installed locally if you want to *run* the app
on a USB-connected device or produce a one-off APK on your own machine.

## TL;DR — produce a release APK locally

```bash
# 1) install Flutter (3.19+ / Dart 3.3+) and add it to PATH
#    https://docs.flutter.dev/get-started/install

# 2) one-time bootstrap inside this folder
cd tools/app
flutter create . --project-name x_media_ci_app --platforms=android,linux,windows
flutter pub get

# 3) build the Android 14+ release APK
flutter build apk --release           # default: arm64-v8a + armeabi-v7a
flutter build apk --release --split-per-abi
# Output: build/app/outputs/flutter-apk/app-release.apk
```

## TL;DR — run on a connected device

```bash
# USB-connected Android device with developer options enabled
flutter devices
flutter run -d <device_id>
```

For a phone that should talk to the **x_media CI server on a PC** over the same
WiFi, set the server's URL in the app's **Settings** tab to
`http://<PC_IP>:8765`. For USB-only setups use `adb reverse`:

```bash
adb reverse tcp:8765 tcp:8765
# Settings -> http://127.0.0.1:8765
```

## What's *not* automated in this folder

The `android/`, `linux/`, `windows/` platform directories are intentionally
not committed — they're generated by `flutter create .` so that the
project tracks a single source of truth (`lib/`, `pubspec.yaml`).

For CI builds see `.github/workflows/release.yml` (ubuntu-latest) which
builds + uploads the Android APK as a release artifact.

## Minimum SDK levels (Android 14+)

Default Flutter `compileSdk` is high enough for API 34 / Android 14.
If you need to bump:

```yaml
# android/app/build.gradle (or build.gradle.kts)
android {
  defaultConfig {
    minSdkVersion 24   # 14+ is API 34
    targetSdkVersion 34
  }
}
```
